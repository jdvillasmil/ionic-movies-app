    - If reviews is empty: ion-note "No hay reseñas aún"
    - If reviews has items: an ion-card per review showing mediaTitle, score (e.g., "8/10"), and a preview of the text (truncated at 100 chars with "...")

    Note: reviews collection may not exist yet (Phase 4 creates it). Guard against empty results gracefully — the query returns an empty array, render the empty state.

    ACCOUNT DELETION:
    Add a "Eliminar cuenta" button below the logout button (ion-button color="danger", fill="outline", expand="block").

    Deletion flow:
    1. Show AlertController confirm dialog (2-step as per user decision):
       header: 'Eliminar cuenta'
       message: 'Esta acción es irreversible. Se eliminarán tu cuenta y todas tus reseñas.'
       buttons: ['Cancelar' (cancel), 'Eliminar' (destructive)]
    2. If confirmed:
       a. Get uid from authService.currentUser!.uid
       b. await userService.deleteUserAndReviews(uid)   // Firestore first
       c. await firstValueFrom(authService.deleteCurrentUser())   // Auth record second
       d. navigate to /login
    3. On auth/requires-recent-login error: show toast "Por seguridad, cierra sesión e inicia de nuevo para eliminar tu cuenta."
    4. On any other error: show toast "Error al eliminar la cuenta. Intenta de nuevo."

    CRITICAL ORDERING: Always delete Firestore data (step b) BEFORE the Auth record (step c). Once the Auth record is deleted, Firestore security rules block any further operations under that UID.

    AlertController: import from @ionic/angular (not from standalone). Already injected in Task 1.
    ToastController: Already injected in Task 1.
