    Export these types at top of file (downstream plans import from here):
    ```typescript
    export interface MediaItem {
      id: number;
      mediaType: 'movie' | 'tv';
      title: string;
      year: string;
      posterPath: string | null;
      voteAverage: number;
      genreIds: number[];
    }

    export interface FilterState {
      genreIds: number[];
      minScore: number;
      yearFrom: string | null;
      yearTo: string | null;
      sortBy: 'popularity.desc' | 'vote_average.desc' | 'primary_release_date.desc';
    }

    export interface TmdbPageResult {
      results: MediaItem[];
      totalPages: number;
    }

    export interface Genre {
      id: number;
      name: string;
    }
    ```

    TmdbService implementation rules:
    - `@Injectable({ providedIn: 'root' })`
    - `private http = inject(HttpClient)` and `private firestore = inject(Firestore)`
    - `private readonly BASE = 'https://api.themoviedb.org/3'`
    - `private readonly CACHE_TTL_MS = 30 * 60 * 1000` (30 min)
    - `private genreCache: Genre[] = []` (in-memory, not Firestore)

    Methods to implement:
    1. `getTrending(page = 1): Observable<TmdbPageResult>` — calls `/3/trending/all/week`, cache key `trending_p${page}`, filters person results, normalizes.
    2. `searchMulti(query: string, page = 1): Observable<TmdbPageResult>` — calls `/3/search/multi`, cache key `search_${encodeURIComponent(query.toLowerCase())}_p${page}`, filters person results, normalizes.
    3. `discoverMedia(filters: FilterState, page = 1): Observable<TmdbPageResult>` — calls `/3/discover/movie` AND `/3/discover/tv` in parallel via forkJoin; builds SEPARATE param objects (movie uses `primary_release_date.gte/lte`, TV uses `first_air_date.gte/lte`); merges results array; re-sorts merged array client-side by filters.sortBy (by voteAverage desc or year desc); cache key built from all filter params + page.
    4. `getGenres(): Observable<Genre[]>` — calls both `/3/genre/movie/list` and `/3/genre/tv/list`, merges union by deduplicating on id, stores in `this.genreCache`, returns from cache if already loaded.
    5. `private cacheFirst<T>(cacheKey: string, fetchFn: () => Observable<T>): Promise<T>` — checks `tmdb_cache/${cacheKey}` doc; if exists and cachedAt within TTL returns `data.results`; else calls fetchFn, stores `{ cachedAt: Date.now(), results }` in Firestore, returns result.
    6. Private helper `normalize(item: any, forcedMediaType?: 'movie' | 'tv'): MediaItem` — maps raw TMDB fields.
    7. Private helper `buildDiscoverMovieParams(filters, page)` and `buildDiscoverTvParams(filters, page)` — separate param builders to avoid date field confusion (critical: TV uses first_air_date, not primary_release_date; TV sort uses first_air_date.desc not primary_release_date.desc when sortBy is 'primary_release_date.desc').
    8. Discover cache key format: `discover_${JSON.stringify({...filters, page})}` — include all filter params.

    All public methods return `Observable<TmdbPageResult>` (wrap Promise from cacheFirst in `from()`).

    After writing service, write spec file for it. Use HttpClientTestingModule + HttpTestingController for TMDB HTTP interception. Mock Firestore using jasmine.createSpyObj for getDoc/setDoc. Test the behaviors listed in the `<behavior>` block above.
